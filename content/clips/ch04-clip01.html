🚀 Antigravity는 "내 컴퓨터 안에 들어온 AI 실무자"입니다. 채팅으로 일을 시키면 에이전트가 직접 내 파일을 정리하고, 자료를 찾아오고, 문서·웹앱을 만들어 줍니다. 이 섹션은 강의 중 놓친 부분을 혼자 복습하거나, 교육 전 미리 감을 잡고 싶을 때 보는 참고 자료입니다.
2026년 Google I/O에서 발표된 Antigravity 2.0은 단순한 코드 에디터를 넘어, 에이전트에게 일을 맡기는 것을 중심에 둔 도구로 바뀌었습니다. 코드를 한 줄도 몰라도, 한국어로 목표만 말하면 됩니다.
🤔 왜 Antigravity인가 — 평소 쓰던 AI와 "근본적으로" 다릅니다

가장 많이 나오는 질문이 "ChatGPT·Gemini랑 뭐가 달라요?"입니다. 답은 — 쓰는 방식이 조금 다른 게 아니라, 동작 방식과 할 수 있는 일의 범위가 근본적으로 다릅니다.

구분웹 채팅 AI (ChatGPT · Claude · Gemini 웹)에이전틱 AI (Antigravity)
무엇을 하나물어보면 답(텍스트·이미지)을 돌려준다목표를 주면 일 자체를 직접 수행한다
내 컴퓨터·파일손대지 못함 — 결과를 내가 복사해 옮겨야 함내 폴더의 파일을 직접 읽고·만들고·정리한다
도구 사용대화창 안에서만 (텍스트 주고받기)웹 검색·브라우저·파일·터미널을 스스로 골라 쓴다
끝나는 방식답을 받으면 그다음은 내가 이어서 작업계획 → 실행 → 결과물(보고서·엑셀·웹앱)까지 알아서
비유똑똑한 상담원 (물어보면 알려줌)내 옆자리 실무자 (시키면 해놓음)
📈 이게 지금 AI의 흐름입니다. 2026년 AI는 '대화(chat)'에서 '위임(agent)'으로 넘어가고 있습니다. 질문에 답하던 AI가, 이제 도구를 쥐고 일을 끝까지 해내는 단계로 진화했습니다. 오늘은 그 변화를 — '에이전틱 AI'를 — 직접 손으로 한 번 경험해 보는 자리입니다.
❓ "대화형 Gemini로도 똑같이 되지 않나요?" — 자주 나온 질문
좋은 질문입니다. Antigravity도 엔진(두뇌)은 같은 Gemini를 씁니다. 그래서 "Gemini 안의 Gemini 아니냐"는 느낌이 들 수 있어요. 차이는 '두뇌'가 아니라 '손'에 있습니다.
  • 대화형 Gemini = 말하는 두뇌. 프롬프트가 아무리 완벽해도 결과는 '텍스트·코드 덩어리'이고, 그걸 내가 복사해 파일로 만들고·실행하고·고치는 건 사람 몫입니다.
  • Antigravity = 같은 두뇌 + 손(파일·실행·도구·반복 루프). 목표만 주면 파일을 직접 만들고 → 실행해 보고 → 오류를 스스로 고치는 과정까지 알아서 돕니다.
👉 즉 "프롬프팅만 잘하면 대화형 Gemini로도 동일"한 건 '한 번의 답'까지입니다. 여러 단계를 직접 실행·반복해 결과물을 완성하는 일은 에이전트(Antigravity)의 영역이에요. 같은 엔진이라도 '할 수 있는 일의 범위'가 달라지는 이유입니다.
🎯 그런데 왜 하필 Antigravity인가 — 다른 에이전트 도구도 많은데

"내 컴퓨터에서 일하는 AI"는 Antigravity 말고도 Claude Code·Codex 등 여럿입니다. 그런데도 오늘 Antigravity를 고른 이유가 있습니다.

💳 구글 구독 하나로 끝
오늘 발급받은 Google AI Pro 구독에 포함 — 별도 결제·복잡한 API 설정 없이 바로 사용.
🔋 사용량(quota)이 넉넉
같은 구독값 대비 쓸 수 있는 양이 넉넉한 편 — 실습 중 한도에 덜 막힙니다. (Claude Code 등은 더 빨리 소진되는 편)
🖥️ GUI가 친화적
새까만 터미널이나 복잡한 코드 화면 없이, 채팅 한 칸으로. 개발 도구처럼 답답하지 않습니다.
🗣️ 한국어로 그냥 시키면 됨
명령어를 외울 필요 없이 평소 말투로. 못 하는 건 에이전트가 알아서 방법을 찾습니다.
정리하면 — "내 컴퓨터에서 일하는 에이전트 AI"라는 큰 흐름을, 가장 부담 없이 체험할 수 있는 도구가 오늘의 Antigravity입니다. 중요한 건 특정 제품이 아니라, 이런 방식으로 일이 바뀌고 있다는 감각입니다.
📦 무엇이 달라졌나 — "IDE"에서 "에이전트 도구 브랜드"로

기존 Antigravity는 코드를 편집하는 IDE 한 가지였습니다. 2.0부터는 같은 에이전트를 여러 형태로 쓸 수 있는 브랜드로 확장됐습니다.

형태설명
Antigravity 앱
오늘 사용		설치형 프로그램. 채팅으로 에이전트에게 일을 맡기는 오늘 실습의 메인 도구입니다.
Antigravity CLI터미널에서 같은 에이전트를 부르는 명령줄 버전 (개발자용 — 오늘은 다루지 않습니다).
SDK자기 프로그램 안에 에이전트를 끼워 넣는 개발 키트 (참고).
🪄 가장 큰 변화 — "코드 편집기가 사라졌다"

2.0의 첫인상은 화면이 단순해진 것입니다. 예전의 복잡한 코드 편집 공간·터미널이 기본 화면에서 빠지고, 에이전트와 대화하는 채팅이 중심이 됐습니다. "코드를 보는 곳"이 아니라 "일을 맡기는 곳"으로 성격이 분명해진 셈입니다.

기존 Workspace라는 이름도 Project로 바뀌었습니다. 코드를 직접 안 봐도 됩니다 — 에이전트가 만든 결과(보고서·엑셀·웹앱)를 눈으로 확인하고 코멘트만 하면 충분합니다.
🧠 어떤 AI가 일하나 — 모델 선택

대화 입력창 위의 모델 선택 버튼에서 일을 시킬 AI를 고릅니다. 기본은 빠르고 똑똑한 Gemini 3.5 Flash이며, 더 깊은 추론이 필요하면 Gemini 3.1 Pro, 그리고 Claude·GPT-OSS 계열도 선택할 수 있습니다.

⚡ Gemini 3.5 Flash (기본·권장)
빠르고 가벼움. 오늘 실습 대부분은 이걸로 충분합니다. (High/Medium/Low 단계 선택 가능 — Low는 토큰 절약)
🧩 Gemini 3.1 Pro
더 복잡한 추론·기획이 필요할 때. 사용량을 더 많이 쓰므로 한도(quota)가 빨리 줄 수 있습니다.
🤝 Claude · GPT-OSS
취향·작업에 따라 다른 AI도 선택 가능. 오늘은 기본 Gemini로 진행합니다.
💡 오늘의 마음가짐 — "쉬운 자동화로 친해지고 → 내 업무로 발전"
오늘은 누구나 공감할 범용 사무 작업(파일 정리·엑셀·자료조사·간단한 웹앱)으로 Antigravity와 친해진 뒤, 자신이 반복하는 실제 업무를 스킬·하네스로 발전시키는 흐름입니다. 완벽하게 외울 필요 없습니다 — "한 번 만들면 다음엔 안 해도 된다"는 감만 가져가시면 됩니다.

👉 그리고 오늘의 진짜 초점은 '코딩'이나 '앱 만들기' 자체가 아닙니다. 후반의 바이브코딩(웹앱)은 "이런 것도 된다"를 보여주는 일부일 뿐, 핵심은 AI가 내 파일을 다루고·자료를 찾고·결과물까지 만들어 주는 '에이전틱 AI'를 직접 경험하는 것입니다. "대화하는 AI"와 "일을 맡기는 AI"의 차이를 몸으로 느끼고 가시면 성공입니다.
