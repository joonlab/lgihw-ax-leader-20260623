🧩 스킬(Skill)은 "한 번 만들면 다음엔 한 줄 요청으로 끝나는" 재사용 묶음입니다. 실습 ④에서 만든 Gemini API 웹앱의 패턴을 스킬로 등록해 두면, 다음엔 "Gemini로 ○○하는 웹앱 만들어줘" 한마디에 같은 품질의 결과가 자동으로 나옵니다. 이 섹션은 자율 심화 시간에 직접 따라하거나, 끝난 뒤 혼자 복습할 때 보는 참고 자료입니다.
반복하는 작업을 매번 처음부터 설명하지 않도록 스킬로 박제합니다. 핵심은 "코드를 다시 짜는 게 아니라, 한 번 검증된 패턴을 AI에게 기억시키는 것". 공통 소재로 감을 잡은 뒤 본인 실제 업무로 바꿔 적용하는 것이 오늘의 목표입니다.
🤔 스킬이란 — "설명을 매번 반복하지 않는 법"

실습 ④에서 우리는 'Gemini-API_레퍼런스.md'를 작업 폴더에 넣고 "이 패턴대로 회의록 정리기를 만들어줘"라고 시켰습니다. 그런데 비슷한 웹앱을 또 만들 때마다 같은 레퍼런스를 다시 챙기고, 같은 규칙을 다시 설명해야 한다면 번거롭습니다.

스킬은 바로 그 "매번 설명하던 것"을 에이전트의 서랍에 넣어두는 일입니다. 한 번 등록하면, 다음부터는 한 줄 요청만으로 에이전트가 알아서 그 패턴(단일 HTML · 모델 ID · API 키 입력란 · 에러 처리 · JSON 출력)을 꺼내 씁니다.

쉽게 말해 — 실습 = 일을 한 번 시키는 것, 스킬 = 그 일을 시키는 방법을 통째로 저장해 두는 것. 그래서 스킬을 만들면 "다음엔 안 해도 된다"가 성립합니다.
💡 왜 '업무 자동화'보다 '스킬 만들기'가 더 쉬운가

"내 업무를 자동화해줘"는 막막합니다 — 기준이 머릿속에만 있고 말로 풀기 어렵기 때문입니다. 반면 스킬은 '이미 검증된 결과물'을 그대로 본보기로 삼습니다. 기준(참고자료)이 손에 잡히니 AI도 헤매지 않습니다.

구분맨손 자동화 요청스킬로 만들기
기준머릿속에만 있음 (말로 풀기 어려움)참고자료·완성 샘플이 명확
결과 품질그때그때 다름검증된 패턴이라 일정
재사용매번 처음부터 설명한 줄 요청이면 끝
💡 그래서 첫 스킬은 '코드 재사용'부터
막연한 업무 자동화보다, 실습 ④의 Gemini API 패턴처럼 기준이 또렷한 것부터 스킬로 만드는 것이 성공률이 높습니다. 일단 한 번 만들어 "이게 되는구나"를 체감한 뒤, 점점 내 업무 소재로 옮겨가세요.
🛠️ 단계 1 — 레퍼런스를 스킬로 등록하기

실습 ④를 막 끝낸 상태(작업 폴더에 'Gemini-API_레퍼런스.md'가 있고, 회의록 정리기를 만든 직후)에서 아래 프롬프트를 그대로 붙여넣습니다. 에이전트가 레퍼런스의 핵심 패턴을 스킬로 박제하고, "앞으로 어떻게 부르면 되는지"까지 알려줍니다.

📋 [복붙] 심화 ① — 스킬 제작
방금 'Gemini-API_레퍼런스.md'를 참고해서 AI 웹앱을 만들었잖아.
앞으로 이런 'Gemini API로 ○○하는 웹앱 만들기' 작업을 반복할 거야.

이 레퍼런스를 스킬(skill)로 등록해줘.
그래서 다음에 내가 'Gemini로 △△하는 웹앱 만들어줘'라고만 하면,
이 레퍼런스의 패턴(단일 HTML, gemini-3.5-flash 모델, API 키 입력란, 에러 처리, JSON 출력 등)을
자동으로 따라서 만들 수 있게 해줘.

스킬을 어떻게 만들었는지, 그리고 앞으로 어떻게 부르면 되는지도 설명해줘.
에이전트는 레퍼런스를 읽고 스킬 파일(설명 + 패턴 규칙)을 만들어 저장합니다. 끝나면 "이렇게 만들었고, 다음엔 이렇게 부르면 됩니다"라는 안내가 따라옵니다 — 그 호출 방법을 기억해 두세요.
⚡ 단계 2 — 만든 스킬로 새 웹앱을 '한 줄'로

이제 스킬이 등록됐으니, 새 웹앱은 레퍼런스를 다시 챙길 필요 없이 한 줄 요청으로 만들어집니다. '고객 문의 분류기'를 예로 시켜 봅니다 — 같은 패턴(API 키 입력란 · gemini-3.5-flash · 에러 처리)이 자동으로 적용되는지 확인하세요.

📋 [복붙] 심화 ② — 스킬 활용: 고객 문의 분류기
방금 만든 스킬을 사용해서, '고객 문의 분류기' 단일 HTML 웹앱을 만들어줘.

- 입력: 고객 문의 내용
- 출력: 카테고리 / 긴급도 / 추천 답변 (표나 카드 형태로)
- 나머지 규칙(API 키 입력란, gemini-3.5-flash 모델, 에러 처리 등)은 스킬 패턴 그대로 따라줘.

다 만들면 브라우저로 열어서 보여줘.
👀 핵심 체감 포인트 — 두 번째 웹앱은 레퍼런스를 한 번도 언급하지 않았는데도 똑같은 골격으로 나옵니다. 이것이 "한 번 만들면 다음엔 설명 안 해도 된다"는 스킬의 힘입니다.
🎯 자율 실습 — 이번엔 '내 업무'로

공통 소재로 감을 잡았다면, 이제 본인이 앞으로도 반복할 일을 하나 골라 스킬로 만들어 보세요. "내 업무에서 매번 똑같이 하는 일"을 떠올리는 것이 시작입니다.

📑 반복 보고서 양식
주간·월간 보고서의 고정 양식을 스킬로. 다음엔 "이번 주 보고서 초안"만 말하면 형식 맞춰 나옵니다.
✅ 점검·체크리스트 절차
매번 같은 순서로 확인하는 점검 절차를 스킬로. 누락 없이 같은 기준으로 검토하게 됩니다.
🔎 자료조사 흐름
실습 ③ 같은 조사 → 출처 포함 보고서 흐름을 스킬로. 주제만 바꿔 같은 품질로 반복 생산.
⚠️ 자율 실습 팁 — 기준이 또렷한 것부터
처음부터 거창한 업무를 스킬로 만들려 하면 막힙니다. 참고할 완성 샘플이나 명확한 양식이 있는 일부터 시작하세요. 그리고 만든 스킬은 한 번 써보며 결과를 검증한 뒤, 부족한 규칙을 조금씩 보강하면 됩니다.
✅ 한 줄 정리
스킬 = 재사용. "한 번 만들면 다음엔 설명 안 해도 된다"가 핵심 메시지입니다. 오늘은 기준이 명확한 코드 패턴(④ Gemini API)으로 감을 잡고, 자율 실습에서 내 업무로 한 걸음 옮겨 가세요.
