2. 자기 노트북 만들기 — 소스 확보 3가지 방법

Step 1. notebooklm.google.com 접속 → "새로 만들기" 클릭 → 빈 노트북 생성

Step 2. 아래 세 가지 방법 중 편한 것을 골라 (또는 조합해) 소스를 추가합니다. 보통 4~6개의 소스가 모이면 결과물 품질이 안정적입니다.

방법 A · 내 자료 첨부
평소 가지고 있던 PDF·워드·PPT를 올린다
업무 보고서, 회의록, 강의 노트, 책 PDF, 사내 자료 등 무엇이든 OK. 드래그앤드롭하거나 '+ 소스 추가' 버튼으로 업로드.
지원 형식: PDF · DOCX · PPTX · TXT · MD · 구글 문서 · 웹 URL · YouTube · 이미지 · 오디오
방법 B · NotebookLM 검색
Fast Research / Deep Research로 키워드 검색
소스 패널 상단의 "웹에서 새 소스를 검색하세요" 검색창에 관심 주제·키워드 입력 → 옆 드롭다운에서 Fast Research(빠른 결과) 또는 Deep Research(심층 보고서) 선택 → 화살표(→) 클릭. NotebookLM이 자동으로 관련 웹 자료를 찾아 소스로 추가합니다.
Fast vs Deep: Fast는 빠르게 몇 개 자료, Deep은 더 오래 걸리는 대신 심층 보고서 형태로 결과 제공.
예시 키워드: "2026 생성형 AI 업무 활용 트렌드", "AI 고객 응대 도입 사례", "에이전틱 AI 기업 도입"
방법 C · 이미지/로고 추가 (선택)
브랜드 톤을 입히려면 로고·디자인 자료도 함께
자기 회사 로고나 참고 디자인을 적용하고 싶다면, 구글 이미지 검색 등에서 로고를 직접 다운받아 추가 소스로 첨부합니다. AI가 그 톤을 참고해 결과물을 만들어 줍니다.
팁: "LG 로고", "LG 브랜드 가이드", "LG 브랜드 컬러" 같은 키워드로 웹 검색 → PNG 저장 → NotebookLM에 첨부
소스 고르는 팁
한 노트북 안의 소스들이 같은 주제·관점이면 결과물이 일관됩니다. 너무 광범위한 자료를 섞으면 결과물도 산만해집니다. 처음에는 한 가지 주제(예: "에이전틱 AI 기업 도입")로 4~6개 소스를 모아 시작하시고, 익숙해지면 다양한 관점을 의도적으로 섞어보세요.
🔎 방법 B 자세히 — Fast Research vs Deep Research

두 모드는 속도·소스 수·결과 형태가 다릅니다. 같은 검색창에서 드롭다운만 바꾸면 됩니다.

비교 항목Fast ResearchDeep Research
소요 시간약 30초~1분약 3~5분
수집 소스 수몇 개의 핵심 자료수십 개의 다양한 자료
결과 형태빠른 자료 모음 (소스 추가)심층 보고서 + 풍부한 자료
적합한 상황빠르게 감 잡기, 키워드 탐색, 회의 직전 보강본격 분석·기획 자료 작성, 경쟁사·시장 심층 조사
드는 비용(시간)가볍게 여러 번 시도 가능한 번에 깊이 — 좋은 키워드 한 방이 효율적
Fast Research 키워드 예시
2026 생성형 AI 업무 활용 트렌드
Deep Research 주제 예시
생성형 AI를 업무에 도입한 국내외 기업 사례를 심층 조사해 주세요. 어떤 업무(마케팅·고객응대·문서작업 등)에 적용했는지, 도입 효과와 한계, 도입 시 주의할 점을 포함해 주세요.
언제 Fast, 언제 Deep?
Fast 추천: "이 주제 자료가 어떤 게 있는지 일단 본다" / "회의 5분 전에 빠른 보강" / 키워드를 여러 개로 바꿔가며 탐색.
Deep 추천: "이걸로 보고서·기획서를 쓴다" / "경쟁사 비교를 정확히 해야 한다" / 신중하게 한 번에 깊은 결과.
조합 팁: Fast로 키워드 감을 잡고, 좋은 키워드가 보이면 같은 노트북 안에서 Deep으로 한 번 더 — 두 종류 자료가 함께 쌓여 분석 품질이 올라갑니다.

인포그래픽 & 슬라이드: 기본 vs 커스터마이징 비교

인포그래픽과 슬라이드는 각각 두 가지 버전을 만들어 비교합니다. 같은 소스라도 어떤 컨텍스트를 주느냐에 따라 결과물이 어떻게 달라지는지를 직접 확인할 수 있습니다.

Version A: 기본 생성
1. 스튜디오 패널에서 "슬라이드 자료" 클릭
2. 아무 커스터마이징 없이 바로 생성
3. AI가 자동으로 구성한 기본 결과물 확인
Version B: 커스터마이징 적용
1. 연필 아이콘(커스터마이징) 클릭
2. 아래 프롬프트 예시를 참고해 자기 의도대로 작성
3. 커스터마이징이 반영된 결과물 확인
커스터마이징할 수 있는 요소
· 청중(누가 볼 것인가) — 임원·실무자·고객·일반 대중·신입 사원 등
· 강조점(무엇을 부각할 것인가) — 데이터·핵심 메시지·실행 액션·리스크·기회 등
· 레이아웃 — 표 중심 / 다이어그램 중심 / 한 장 요약 / 단계별 시리즈
· 디자인·색상 — 미니멀·플랫·일러스트·다크모드 / 핑크·블루·그레이 톤
· 톤앤매너 — 격식체 보고용 · 친근한 설명체 · 짧고 강한 카피
· 분량 — 5장 / 10장 / 20장 등 슬라이드 수 명시
· 브랜드 자료 — 방법 C로 첨부한 로고·디자인 가이드를 참고하라고 명시
커스터마이징 프롬프트 예시 — 자기 의도에 맞게 수정해서 사용
청중: [예: 상사·경영진 / 실무 동료 / 신입 직원 / 고객] 강조점: [예: 핵심 데이터 3개 / 실행 액션 5단계 / 핵심 차별점] 레이아웃: [예: 한 장에 한 메시지 / 표 중심 / 인포그래픽 다이어그램 위주] 디자인·색상: [예: 핑크 포인트의 미니멀 플랫 / 다크 모드 / 브랜드 핑크(#A50034)] 톤앤매너: [예: 격식체 보고용 / 친근한 설명체] 분량: [예: 슬라이드 8장 / 인포그래픽 1장] 브랜드 자료: 첨부한 로고/디자인 이미지를 참고해 일관된 톤을 적용해 주세요. 추가 지시: 각 페이지마다 [핵심 메시지 한 줄]을 상단에 강조해 주세요.

대괄호 안 예시는 자기 도메인·의도에 맞게 바꿔 쓰세요. 인포그래픽과 슬라이드 모두 같은 프롬프트를 활용할 수 있습니다.

4. 마인드맵 탐색

마인드맵이 가장 먼저 완성되므로, 다른 콘텐츠가 생성되는 동안 마인드맵을 먼저 살펴봅니다.

  • 구조 파악: 마인드맵의 각 노드를 펼쳐나가며 보고서의 전체 구조를 한눈에 파악합니다
  • 키워드 탐색: 특정 키워드가 궁금하다면 해당 노드를 클릭하세요
  • 채팅 연동: 노드를 클릭하면 채팅창에서 바로 질문이 가능합니다
핵심 기능: 마인드맵에서 궁금한 키워드를 클릭하면 채팅창에 해당 내용이 자동으로 질문됩니다. AI가 소스를 기반으로 즉시 답변합니다.

마인드맵 출력 예시 — 노드를 클릭하면 해당 키워드에 대해 AI에게 바로 질문할 수 있습니다

5. 인포그래픽 & 슬라이드 리뷰

생성이 완료되면 기본 버전(Version A)과 커스터마이징 버전(Version B)을 비교해 보세요.

비교 항목Version A (기본)Version B (커스터마이징)
디자인 톤AI 기본 스타일지정한 색상·로고·디자인 톤 반영
관점소스 원문 중심 요약지정한 청중·강조점에 맞춰 재구성
레이아웃AI 자동 구성지정한 레이아웃·분량으로 정렬

같은 소스라도 어떤 컨텍스트와 의도를 주느냐에 따라 결과물의 완성도와 활용도가 달라집니다.

이미지를 소스로 활용하면
로고·디자인 가이드라인·참고 이미지를 소스로 함께 넣으면 AI가 해당 스타일을 참고해 일관된 톤의 결과물을 생성합니다. 자기 회사 로고를 첨부했다면 "첨부한 로고를 참고해 색상과 폰트 톤을 맞춰 주세요"처럼 명시적으로 지시하면 더 잘 반영됩니다.
6. 팟캐스트 커스터마이징 (시간 여유 시)

팟캐스트(AI 오디오 오버뷰)도 커스터마이징이 가능합니다. 기본 생성 버전을 들어본 후, 아래 프롬프트 예시를 자기 의도에 맞게 수정해 맞춤형 팟캐스트를 만들어 보세요.

팟캐스트 커스터마이징 프롬프트 예시
대화 형식: [예: 두 명의 전문가 대담 / 인터뷰어와 게스트 / 1인 모놀로그] 청중: [예: 같은 팀 동료 / 일반 대중 / 출퇴근 중 듣는 직장인] 관점: [예: 실무 적용 중심 / 기술적 디테일 / 사례 비교] 톤앤매너: [예: 격식 있는 분석 / 친근한 수다 / 진지한 다큐] 분량: [예: 8~12분] 강조: 핵심 데이터·통계는 자연스럽게 대화에 녹여서 설명해 주세요. 어려운 용어는 한 번씩 풀어 주세요.
동시 생성 활용 팁
NotebookLM에서는 여러 종류의 출력물을 동시에 생성할 수 있습니다. 팟캐스트가 생성되는 동안 이미 완성된 마인드맵이나 슬라이드를 탐색하세요. 오디오 오버뷰를 들으면서 마인드맵을 탐색하는 멀티태스킹도 가능합니다.
실습 중 자주 막히는 지점
· 생성이 오래 걸려 멈춘 것 같을 때 — 팟캐스트·영상은 원래 수 분이 걸립니다. 먼저 생성을 걸어두고 그동안 마인드맵·Q&A를 진행하세요. 점심·휴식 시간을 생성 대기 버퍼로 쓰면 좋습니다.
· 결과물을 패들렛/링크로 공유했는데 "액세스 권한 없음"이 뜰 때 — 우측 상단 공유 버튼을 누르면 '노트북 액세스'가 기본값 "제한됨(액세스 권한이 있는 사용자만 링크로 열 수 있음)"으로 되어 있습니다. 드롭다운을 "링크가 있는 모든 사용자"로 바꿔야 상대가 링크로 열 수 있습니다. 그 뒤 '링크 복사'로 주소를 전달하세요. (특정인에게만 줄 때는 '사용자 및 그룹 추가'에 이메일을 입력합니다.)
· 리서치 결과에 원하던 소스가 빠졌을 때 — 수집된 소스 목록에서 체크박스로 직접 포함·제외할 수 있습니다.
