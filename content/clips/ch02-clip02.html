AI에게 효과적으로 지시하는 핵심 프레임워크와 프롬프트 작성 기법을 배웁니다.
AI에게 효과적으로 지시하는 7가지 요소 (Delegation Canvas)

좋은 프롬프트의 본질은 "AI에게 업무를 위임하는 지시서"를 잘 작성하는 것입니다.

#요소프롬프트 예시업무 활용 팁
1Role (역할)"당신은 10년 차 마케팅 기획자입니다."기획자 / 데이터 분석가 / 보고서 작성자
2Goal (목표)"아래 자료를 분석하여 핵심 이슈 3가지를 추출해주세요."분석 / 비교 / 요약 / 작성 / 번역
3Context (맥락)"다음 주 팀 회의 안건입니다. 첨부한 자료를 기반으로 분석해주세요."업무 배경 / 회의 안건 / 고객 상황
4Constraints (제약)"민감 정보는 제외하고, 2페이지 이내 격식체로 작성해주세요."민감정보 제외 / 2페이지 이내 / 격식체
5Output (형식)"표 형태로 옵션 3개를 비교 매트릭스로 정리해주세요."표 + 매트릭스 / 1페이지 요약 / 옵션 3개
6Evaluation (평가)"각 옵션에 기대 효과와 리스크 수준을 반드시 포함해주세요."기대효과 포함 / 리스크 분석 / 비교
7Questions (선제 질문)"분석 전에 먼저 물어봐야 할 사항이 있으면 질문해주세요."예산 범위? 우선순위? 대상 고객?
실전 팁
7가지를 모두 포함할 필요는 없습니다. 처음엔 Role + Goal + Output 3개만 넣어도 답변 품질이 눈에 띄게 향상됩니다. 익숙해지면 Context와 Constraints를 추가하세요.
프롬프트 엔지니어링 4원칙

Delegation Canvas를 실전에 적용할 때, 특히 중요한 4가지 원칙입니다.

원칙설명활용 예시
Format (형식)출력 형태를 구체적으로 지정하면 AI가 즉시 활용 가능한 결과를 생성합니다"표 형태로", "A4 1장 분량으로", "불릿 포인트로"
Role (역할)AI에게 전문가 역할을 부여하면 관점과 깊이가 달라집니다"마케팅 기획자로서", "데이터 분석가로서", "고객 상담사로서"
Context (맥락)파일 첨부로 풍부한 배경 정보를 제공하면 정확도가 크게 향상됩니다회의록 첨부, 보고서 첨부, 고객 문의 내역 첨부
Step (단계)하나의 긴 프롬프트보다, 대화를 이어가며 단계적으로 깊이를 더합니다요약 → 이슈 추출 → 개선안 제안 → 시나리오 분석
4원칙 적용 순서
Format → Role → Context → Step 순서로 프롬프트를 구성하면 자연스럽게 좋은 결과를 얻을 수 있습니다. 처음엔 Format과 Role만 적용해도 답변 품질이 크게 달라집니다.
컨텍스트 엔지니어링 연결
프롬프트 엔지니어링 4원칙은 컨텍스트 엔지니어링의 실천 도구입니다. 역할·맥락·형식·단계적 사고를 통해 AI에게 더 풍부한 컨텍스트를 제공하는 것이 곧 더 좋은 결과로 이어집니다.

After (효과적)
당신은 글로벌 전략 컨설팅 펌의 시니어 파트너입니다. 분야: 내가 일하는 산업·업무 분야 분석 관점: 실무 의사결정 지원용 2025~2026년 AI 산업의 주요 트렌드 Top 5를 다음 형식으로 정리해 주세요: - 각 트렌드별: 트렌드명 / 현황 / 우리 업무에 미치는 영향 / 대응 시급도(상/중/하) - 표 형태로 정리 - 팀 회의에서 바로 언급할 수 있는 수준의 인사이트 포함
자주 막히는 지점
· 답변이 너무 일반적일 때 — 역할(Role)과 맥락(Context)이 비어 있는 경우가 대부분입니다. "당신은 ○○ 전문가입니다" 한 줄, 그리고 관련 자료 첨부만 추가해도 답이 확 달라집니다.
· 형식이 마음에 안 들 때 — 같은 대화에서 "표로 다시", "3줄 요약 먼저"처럼 후속으로 고치면 됩니다. 처음부터 완벽하게 쓰려 하지 마세요.
· 답이 점점 이상해질 때 — 한 대화에 주제를 너무 많이 섞으면 맥락이 오염됩니다. 주제가 바뀌면 새 채팅에서 다시 시작하는 것이 더 빠릅니다.
