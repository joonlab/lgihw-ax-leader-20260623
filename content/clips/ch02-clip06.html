자기만의 Gems(나만의 AI 어시스턴트)를 직접 만들어 보는 시간입니다. 아래 가이드와 예시는 참고용이며, 평소 자기 업무·관심사에 맞는 Gems를 자유롭게 설계해보세요.
이 실습의 목표
1. 자기 업무·관심사에 맞는 Gems 주제를 정합니다 (아이디어 큐레이션 참고)
2. 시스템 인스트럭션(요청 사항)을 작성합니다 — 직접 작성하거나 AI에게 작성을 부탁하거나, 요술봉 버튼을 활용합니다
3. 생성한 Gems와 테스트 대화를 나누며 의도대로 작동하는지 검증합니다
💡 Gems 아이디어 큐레이션 — 무엇을 만들어볼까?

평소 자기가 자주 하는 일, 관심 있는 주제, AI에게 잘 시키는 것 중 하나를 골라보세요. 아래는 시작점으로 활용할 수 있는 아이디어입니다.

✍️ Gems 지침(요청 사항) 작성 팁

시스템 인스트럭션을 처음부터 완벽하게 쓸 필요는 없습니다. 다음 세 가지 방법 중 편한 것을 골라 시작하세요.

방법 1. 직접 작성
아래 4원칙(역할·맥락·형식·단계적 사고)을 떠올리며 직접 씁니다. 짧게 시작해도 OK — 테스트하면서 개선하면 됩니다.
방법 2. AI에게 부탁하기 (추천)
Gemini 일반 채팅이나 다른 AI에게 평소처럼 부탁합니다. 예: "긴 보고서를 핵심만 뽑아 요약해 주는 Gems를 만들고 싶어. 시스템 인스트럭션을 잘 짜서 Markdown으로 보여줘." 결과를 그대로 복사해서 Gems의 '요청 사항' 칸에 붙여넣으면 됩니다.
방법 3. 요술봉 버튼 (간단)
Gems 편집 화면의 '요청 사항' 입력창 안에 요술봉(✨) 버튼이 있습니다. 한 두 줄만 적어두고 이 버튼을 누르면 Gemini가 알아서 프롬프트를 고도화해 줍니다. 가장 빠른 방법.
방법 2 — AI에게 부탁할 때 쓸 수 있는 메타 프롬프트 예시
나는 [내가 만들고 싶은 Gems 주제, 예: 긴 보고서를 핵심만 뽑아 요약]을 도와주는 Gems(Gemini 맞춤 어시스턴트)를 만들고 싶어. 다음 4원칙을 반영해서 시스템 인스트럭션을 작성해줘: 1. 역할: 어떤 전문가 페르소나로 답해야 하는지 2. 맥락: 누가·왜 사용하는지, 어떤 배경을 알아야 하는지 3. 출력 형식: 어떤 구조(표/요약/리스트)로 답해야 하는지 4. 단계적 사고: 답하기 전 어떤 순서로 분석해야 하는지 추가 조건: - 한국어로 작성 - 격식체 사용 (혹은 친근한 반말 — 골라줘) - 잘못된 정보를 만들지 않도록 "확인 필요" 표기 규칙 포함 - 마크다운 헤딩(##) 구조로 보기 좋게 정리해줘
아래는 예시 — 그대로 따라해도 되고, 자기 Gems로 변형해도 됩니다
아래 Step 1~3은 "보고서 요약 어시스턴트"라는 예시 Gems를 만드는 흐름입니다. 누구나 쓰는 공통 업무(긴 문서를 핵심만 뽑아 정리)를 다루므로 직무와 무관하게 따라 할 수 있습니다. 처음 만드시는 분은 그대로 따라해보고, 익숙해지면 위 큐레이션에서 고른 자기 주제로 바꿔서 다시 만들어 보세요. 이름·설명·요청 사항·테스트 질문은 모두 자기 도메인에 맞게 바꿔 입력하면 됩니다.
Step 1: Gems 생성하기 (공통 절차)

Gem 관리자 페이지 — Google 사전 제작 Gems와 내 Gems 목록이 표시됩니다. 우측의 '+ 새 Gem' 버튼을 클릭합니다

  • 1. gemini.google.com 접속
  • 2. 좌측 사이드바 → "Gems" 클릭 → Gem 관리자 페이지 이동
  • 3. "+ 새 Gem" 클릭
  • 4. 아래 이름, 설명을 복사하여 입력하고, 시스템 인스트럭션을 붙여넣기
  • 5. "저장" 클릭
Name (이름)
보고서 요약 어시스턴트
Description (설명)
긴 보고서·회의록·기사를 업로드하면 핵심만 뽑아 구조화된 요약과 실행 액션으로 정리해 주는 AI 어시스턴트. 직무와 무관하게 "긴 문서 빠르게 파악하기"에 활용합니다.

새 Gem 생성 화면 — 이름, 설명, 요청 사항(시스템 인스트럭션), 기본 도구, 지식 필드를 작성합니다

Step 2: 요청 사항 입력 (예시 — 자기 Gems로 변형 가능)

아래는 "AX 전략 회의 어시스턴트"의 시스템 인스트럭션 예시입니다. 그대로 복사·붙여넣기 해도 되고, 위에서 고른 자기 Gems 주제에 맞춰 바꿔도 됩니다. 직접 작성이 어렵다면 요술봉(✨) 버튼이나 Gemini 일반 채팅에 부탁해서 초안을 받아오세요.

시스템 인스트럭션 입력 완료 — '요청 사항' 필드에 인스트럭션을 붙여넣고, 우측 미리보기에서 결과를 확인합니다

Instructions (요청 사항)
당신은 긴 문서를 핵심만 뽑아 정리해 주는 보고서 요약 어시스턴트입니다. ## 역할 정의 사용자가 업로드하거나 붙여넣은 보고서·회의록·기사·메일 스레드 등 긴 텍스트를 읽고, 핵심을 구조화된 요약과 실행 가능한 액션으로 정리합니다. 어떤 직무·산업이든 "긴 문서를 빠르게 파악하고 다음 행동을 결정"하도록 돕는 것이 목표입니다. ## 커뮤니케이션 스타일 1. **격식체 사용**: 비즈니스 한국어를 사용합니다. ("~입니다", "~하겠습니다") 2. **결론 우선**: 핵심 메시지를 맨 앞에 제시하고, 세부 내용은 그 뒤에 구조화합니다. 3. **근거 기반**: 요약 내용은 반드시 원문에 있는 내용만 사용하고, 원문에 없는 내용을 추측해 넣지 않습니다. 4. **표와 구조화**: 복잡한 정보는 테이블, 불릿 포인트로 정리합니다. 5. **실행 가능성**: 액션을 제안할 때는 "누가, 언제까지, 무엇을"을 포함합니다. ## 핵심 역량 ### 1. 핵심 요약 - 긴 문서를 3줄 이내 핵심 요약 + 항목별 상세 요약으로 정리 - 문서의 목적, 결론, 주요 주장 순으로 구조화 ### 2. 쟁점·리스크 추출 - 의사결정이 필요한 쟁점, 미결 사항, 주의해야 할 리스크를 자동 추출 - 각 쟁점에 대해 배경과 영향(왜 중요한가)을 함께 설명 ### 3. 실행 액션 제안 - 문서 내용을 바탕으로 당장 실행할 수 있는 액션 3~5가지를 제안 - 각 액션에 우선순위와 "누가/언제까지/무엇을"을 명시 ### 4. 형식 변환 - 같은 내용을 보고용 1페이지 요약, 메일 초안, 발표용 불릿 등 원하는 형식으로 재정리 ## 응답 형식 가이드 ### 요약 요청 시 [핵심 요약] (3줄 이내) [상세 내용] - 항목별 구조화 [쟁점·리스크] - 주의가 필요한 부분 [실행 액션] - 우선순위가 매겨진 액션 아이템 ## 주의사항 - 원문에 없는 내용은 만들어 넣지 않습니다. 추론이 필요하면 "추정" 또는 "원문에 명시되지 않음"으로 표기합니다. - 확인되지 않은 수치·날짜·고유명사는 임의로 채우지 않습니다. - 의사결정을 대신하지 않으며, 판단에 필요한 정리와 옵션을 제공하는 역할입니다.

'저장' 클릭 후 — Gem이 생성되었다는 확인 메시지가 나타납니다. '채팅 시작'을 눌러 바로 테스트할 수 있습니다

Step 3: Gems 테스트 대화 (예시 — 자기 도메인 질문으로 바꿔도 OK)

생성한 Gems가 의도대로 작동하는지 확인합니다. 아래는 "보고서 요약 어시스턴트" 기준 테스트 질문 예시입니다. 자기 Gems를 만드신 분은 평소 그 주제로 자주 묻는 질문 2~3개를 직접 입력해 보세요. 응답 톤·형식·정확도가 의도대로 나오면 성공입니다.

테스트 1: 문서 핵심 요약
다음 회의록을 요약해줘. "이번 주 팀 회의에서는 신규 캠페인 일정이 2주 지연된 점이 보고되었다. 원인은 디자인 시안 확정 지연과 외부 업체 일정 충돌이었다. 대안으로 시안 후보를 2개로 줄여 빠르게 결정하자는 의견과, 일정을 그대로 유지하되 인력을 추가 투입하자는 의견이 나왔다. 다음 회의까지 각 안의 비용을 비교해 오기로 했다."

확인 포인트: 핵심 요약이 3줄 이내로 먼저 나오는지, 쟁점(2개 안)과 후속 액션이 구조화되는지, 격식체를 사용하는지 확인하세요.

테스트 2: 쟁점·리스크 추출
방금 요약한 회의록에서, 의사결정이 필요한 쟁점과 주의해야 할 리스크를 표로 정리해줘. 각 항목이 왜 중요한지도 한 줄씩 설명해줘.

확인 포인트: 비교 표가 포함되는지, 원문에 없는 내용을 지어내지 않는지, 불확실한 부분에 "원문에 명시되지 않음" 등 표기를 하는지 확인하세요.

테스트 3: 실행 액션 정리
이 회의록을 바탕으로, 다음 회의 전까지 우리 팀이 해야 할 액션을 우선순위 순으로 3가지 정리해줘. 각 액션마다 "누가, 언제까지, 무엇을"을 포함해줘.

확인 포인트: 액션이 우선순위 순으로 제시되는지, "누가, 언제까지, 무엇을" 형태로 구체화되는지 확인하세요.

실습 중 자주 막히는 지점
· 지침을 저장했는데 반영이 안 되는 느낌일 때 — 저장 후 새 대화(채팅 시작)를 열어야 지침이 적용됩니다. 기존 대화창에서는 이전 맥락이 섞일 수 있습니다.
· 만든 Gems를 공유할 때 "권한 없음"이 뜰 때 — 공유 설정에서 액세스를 반드시 "링크가 있는 모든 사용자"로 바꿔야 다른 사람이 열 수 있습니다.
· 요술봉(✨) 버튼이 동작하지 않을 때 — Gemini 일반 채팅에 "이런 Gems 지침을 markdown으로 짜줘"라고 부탁해 결과를 복사·붙여넣는 방법으로 우회하세요. 오늘 실습에서는 이 방법을 메인으로 권장합니다.
